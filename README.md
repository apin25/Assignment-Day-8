# Refleksi Penggunaan GitHub dalam Proyek Bootcamp

## Deskripsi
Repository ini berisi kumpulan tugas harian (Day 3â€“Day 7) selama mengikuti Bootcamp Mobile App Development di Dibimbing.  
Setiap folder mewakili satu hari pembelajaran dan dikembangkan di branch terpisah sebelum digabungkan ke `main` melalui Pull Request.

## Workflow yang Digunakan
- Setiap **Day** dibuat pada **branch berbeda**.
- Setiap branch memiliki **issue** dan **pull request** sendiri.
- **Commit message** selalu disertai referensi ke issue (`#1`, `#2`, dst) agar perubahan terdokumentasi dengan baik.
- Semua PR direview sebelum di-merge ke `main`.

## Pembelajaran dari Penggunaan GitHub
Dari pengalaman ini, saya belajar bahwa:
1. **GitHub memudahkan kolaborasi dan dokumentasi.**  
   Dengan fitur branch, pull request, dan issues, setiap perubahan bisa ditinjau dan dikelola dengan aman.
2. **Commit message yang jelas sangat membantu tracking progress.**
3. **Pull request dan review** melatih cara berpikir kolaboratif, bahkan ketika dikerjakan sendiri.
4. **.gitignore** penting untuk menjaga repository tetap bersih dari file build dan konfigurasi lokal.
5. Penggunaan **issue per hari** membuat alur kerja lebih terorganisir dan menyerupai lingkungan kerja profesional.

## Kesimpulan
Git dan GitHub bukan hanya alat untuk menyimpan kode,  
tapi juga untuk melatih kebiasaan kerja yang rapi, kolaboratif, dan terstruktur.  
Selama proyek ini, saya merasa lebih mudah memantau progres dan menghindari kesalahan versi kode.
